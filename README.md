# 网格交易回测工具（Panel）

一个基于 Python + Panel 的网格交易回测应用，支持股票/ETF 历史数据获取、策略回测、K 线可视化、交易记录展示与 CSV 导出。

## 功能概览

- 数据源优先级：`akshare`（A 股/ETF）→ `yfinance`（回退）→ 模拟数据（兜底）
- 默认回测区间：最近半年
- 首笔建仓规则：默认使用当前现金的 `50%` 建仓
- 网格交易规则：
	- 价格上涨达到阈值触发卖出
	- 价格下跌达到阈值触发买入
- 最小/最大持股限制：支持开关，默认均关闭（可选项）
- 图表与表格展示：按左侧用户输入的开始日期和结束日期过滤
- 交易记录支持分页展示与 CSV 下载

## 启动方式

### 方式 1：直接运行 Python 脚本（推荐）

```bash
python .\grid_trading_backtest.py
```

可指定端口：

```bash
python .\grid_trading_backtest.py --port 5007
```

> 当端口被占用时，程序会自动回退到随机可用端口。

### 方式 2：使用 Panel 命令

```bash
panel serve .\grid_trading_backtest.py --show
```

## 参数说明（左侧面板）

- `股票/ETF代码`：如 `510300`、`159915`、`AAPL`
- `开始日期 / 结束日期`：用于数据获取和右侧展示过滤
- `上涨卖出百分比 / 下跌买入百分比`：网格触发阈值
- `手续费`：按百分比输入（如 `0.03` 表示 0.03%）
- `每格交易数量（股）`：网格单次交易手数
- `启用最少持股限制` + `最少持股数`：可选
- `启用最大持股限制` + `最大持股数`：可选
- `网格价格下限 / 上限`：可选价格约束
- `初始资金`：回测初始现金

## 自动优化使用说明（MVP）

左侧面板新增了参数优化区域，可用于自动搜索更优网格参数组合。

### 操作步骤

1. 先设置代码、日期区间和基础参数（尤其是手续费、持仓限制、资金）。
2. 设置 `优化结果 Top-N`（例如 5）。
3. 点击 `⚡ 自动优化参数（MVP）`。
4. 右侧会出现优化结果表（Top-N 组合）。
5. 点击 `✅ 应用最佳参数并回测`，会自动回填最佳参数并重新执行回测。

### 当前优化范围（MVP）

- 卖出阈值（围绕当前值上下浮动）
- 买入阈值（围绕当前值上下浮动）
- 每格交易股数（围绕当前值和步长浮动）

### 结果排序逻辑

优化结果默认按以下优先级排序：

1. 总收益率高优先
2. 最大回撤低优先
3. 交易次数少优先

### 注意事项

- 优化会执行多次回测，数据区间越长、组合越多，耗时越高。
- 最佳参数依赖历史区间，不保证未来表现。
- 建议先在较短区间快速试参，再在目标区间复核。

## 展示与统计说明

- 回测计算：使用所获取到的完整数据执行策略
- K线图与交易表：按左侧日期区间显示
- 回测摘要会展示：
	- 核心收益指标（收益率、回撤、交易次数等）
	- 请求的数据区间
	- 实际返回数据范围（便于判断数据源是否截断）

---

## 数据源限制说明

分钟级数据常受数据源历史窗口限制（例如仅返回近 30~60 天）。

为尽量覆盖用户选择的长区间，本项目在长周期请求时会自动降频（例如由 `5m` 自动调整为 `60m`）以提升可获取范围。

## 常见问题

### 1) 页面空白或无法访问

- 检查终端是否已成功启动并输出访问地址
- 若端口冲突，可改用：`python .\grid_trading_backtest.py --port 5007`

### 2) 图表看起来只有最近一段数据

- 先看摘要中的“实际返回数据范围”
- 若实际返回范围本身较短，通常是上游数据源限制，不是前端过滤问题

### 3) 没有交易记录

- 适当调小网格阈值（上涨卖出/下跌买入百分比）
- 放宽价格区间上限/下限
- 检查最小/最大持股限制是否启用且设置过严
